Writing blog in hacker way - Hexo on Github

Posted on 2016-07-23   |   In hexo

“Wordpress is getting pretty cumbersome” was the first signal when I tried to update it to the latest blogging platform. Since 2014, I’ve got Wordpress 4.2 up and running on Openshift. Just click few buttons and then enter a few details and it’ll be up and running on live site. Unfortunately, I can’t refer my custom domain without a SSL certificate error when in the backend. Openshift requires $20/month Silver plan to add SSL certificate to my application. Additionally, the custom rhcloud.com subdomain that Openshift provided is blocked in mainland China, for this apocalyptic reason I decided to change blogging platform from Wordpress to Ghost.Where to host my Ghost blog and how can I find a FREE VPS as it based on NodeJS, definitely the VPS is not free and the monthly price is expensive. After online research, I got to know that Heroku is a popular choice to host for free. The only problem with Heroku is the setup process extremely complicated and there is too much limitation on parameter configuration. The worst problem for me is Ghost can’t be updated to latest version on Heroku. The reason is similar to Openshift, I have to upgrade my Heroku account from free dynos to hobby (premium account). Actually I’m not satisfied with dashboard of Ghost 0.4.1, it has display problem of responsive design on android mobile. I tried to run latest version of Ghost on Heroku, it failed again and again. Anyway, Heroku is really nice to keep my account alive these years, you can still visit custom subdomain https://allojoy.herokuapp.com/ without problem. What makes me crazy is both rhcloud.com and herokuapp.com are blocked by GFW in mainland China. I was not able to visit them when I was living in Shanghai in the past three years.

The final move to a static blog has provided me with the best experience so far. I found a good alternative to Jekyll called Hexo. In this post I will take you through the basics steps with command line to setup a Hexo blog with a theme and a few plugins to host it on Github pages.

Ready

System

Windows 7 Ultimate SP1, 64-bit Operating System

Requirements

  • Git v1.9.5

  • Node.js v0.10.32

Check installed versions of Git / Node.js / NPM
  • In Windows 7, one of the quickest ways to launch the Command Prompt is to use the Run window (press keyboard shortcuts Win+R to open it). Then, type cmd or cmd.exe and press Enter.

Don’t forget to add the PATH Environment Variable in Windows after running the Node installer.
  • Go to Control Panel > System > Advanced System Settings Then click Environment Variables…
  • Go to the end of the line and add C:\Users\{yourName}\AppData\Roaming\npm On windows 7, I just use %AppData%\npm to make it work for any user.
